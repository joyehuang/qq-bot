# CLAUDE.md

这是一个 QQ 群打卡机器人项目，使用 NapCat 作为 QQ 协议端。

## Git 提交规范

- 使用 Conventional Commit 规范
- 格式：`type(scope): description`
- 示例：`feat(打卡): 新增打卡功能及指令`
- 只需一行，不需要包含 "Generated by Claude Code" 等内容
- **每完成一个功能模块后，自动 commit 并 push 到 GitHub**（push 后 GHA self-hosted runner 会自动跑 `scripts/deploy.sh`，无需手工部署）
- **新增/修改功能时，同步更新 README.md 和 COMMANDS.md**

## 部署规范（重要）

**禁止运行 `docker compose up -d --build` 之类的全量部署命令**。原因：

- napcat 容器持久化的 QQ 登录态对**容器重建**敏感（即便 image 没变，compose 配置 hash 一变就会触发 recreate），重建后必须重新扫码。
- bot 主程序根本不在 docker 里，是宿主机 PM2 跑 ts-node，docker 部署对它无效。

**正确的部署方式：**

| 改动范围 | 部署方法 |
|---|---|
| `src/**`（bot 代码） | `pm2 reload qq-bot --update-env`（或直接 `git push`，GHA 自动 reload） |
| `admin/server/**` | `docker compose pull admin-api && docker compose up -d --no-deps admin-api` |
| `admin/web/**` | `docker compose pull admin-web && docker compose up -d --no-deps admin-web` |
| `prisma/**` | `npx prisma migrate deploy` 后 `pm2 reload qq-bot` |
| `docker-compose.yml` 里 napcat 段 | 手工评估；改完会重建 napcat → 登录态会丢 → 必须先 `docker cp napcat:/app/.config/QQ ./napcat/qq-data` 把会话拷出来 |

**永远不要主动重建 napcat 容器。** 真要动它（升级 image / 改挂载），先跟 user 确认，并在重建前把容器内 `/app/.config/QQ` 拷到宿主 `napcat/qq-data/`。

## 功能发布公告规范

**新功能发布流程：**

1. **功能开发完成后**：
   - 确保代码已通过 TypeScript 类型检查（`npx tsc --noEmit`）
   - 更新相关文档（README.md、COMMANDS.md）
   - 提交代码到 GitHub

2. **生成功能公告**：
   - 使用 `/announce` 命令自动生成公告
   - 命令会读取最近的 Git Commit 记录
   - 通过 AI API 自动排版和润色生成美观的公告

3. **命令使用**：
   - `/announce` - 基于最新 1 条 commit 生成公告
   - `/announce 3` - 基于最近 3 条 commits 生成公告

4. **公告内容自动包含**：
   - 📋 功能变更概述
   - ✨ 新增功能列表
   - 🔧 改进和修复
   - 📖 使用示例（如适用）
   - 🔗 相关文档链接

5. **注意事项**：
   - 确保提交信息遵循 Conventional Commit 规范
   - 提交信息应清晰描述变更内容
   - 公告会自动发送到群组

## 项目结构

- `src/index.ts` - 主程序入口（Bot 主程序）
- `admin/server/` - 管理后台 API 服务（Express + TypeScript）
- `admin/web/` - 管理后台前端（Vue 3 + TypeScript + Vite）
- `prisma/` - 数据库 schema 和迁移
- `napcat/` - NapCat 配置文件
- `COMMANDS.md` - **指令文档（新增/删除指令时必须同步更新）**

## TypeScript 开发规范

### ⚠️ 类型导入规范（重要）

**所有从第三方库导入的 TypeScript 类型定义，必须使用 `import type` 语法！**

❌ **错误写法**：
```typescript
// 错误：将类型和运行时值混合导入
import axios, { AxiosError, AxiosInstance, AxiosRequestConfig, AxiosResponse } from 'axios';
import { createRouter, createWebHistory, RouteRecordRaw } from 'vue-router';
```

✅ **正确写法**：
```typescript
// 正确：分离类型导入
import axios from 'axios';
import type { AxiosError, AxiosInstance, AxiosRequestConfig, AxiosResponse } from 'axios';

import { createRouter, createWebHistory } from 'vue-router';
import type { RouteRecordRaw } from 'vue-router';
```

**原因**：
- Vite 在浏览器运行时会尝试查找导入的导出项
- TypeScript 类型（如 `AxiosInstance`、`RouteRecordRaw`）不会在运行时存在
- 如果不使用 `import type`，会导致运行时错误：`does not provide an export named 'XXX'`

**判断方法**：
- **运行时值**（函数、类、对象）→ 使用普通 `import`
  - 例如：`axios`、`createRouter`、`createWebHistory`、`ref`、`computed`
- **纯类型**（interface、type、泛型参数）→ 使用 `import type`
  - 例如：`AxiosInstance`、`RouteRecordRaw`、`Ref`、`ComputedRef`

## 开发命令

### Bot 主程序
```bash
# 启动开发
npx ts-node src/index.ts

# 数据库迁移
npx prisma migrate dev
```

### 管理后台
```bash
# 启动后端 API（端口 3001）
cd admin/server
npm run dev

# 启动前端（端口 5173/5174）
cd admin/web
npm run dev
```
